[ purpose ]
SCOPE
Structured after Red Canary's annual Threat Detection Report — methodology, prevalence data, and technique-level findings — but scoped down to a single lab instead of an enterprise telemetry pool of 30,000+ confirmed threats. This report doesn't try to reproduce that scale; it applies the same discipline (real data, verified claims, MITRE-mapped findings) to what's actually observable from one self-hosted SOC. Individual incidents get their own write-ups elsewhere in this portfolio (HUNT-002, IRP-001) — this report's job is to synthesize the detection layer itself: what's actually firing, what silently isn't, and why.
[ methodology ]

Detections are generated by Wazuh's rule engine against host-forwarded syslog (sshd auth events) from two agents. Stock SSH ruleset (0095-sshd_rules.xml) handles baseline classification; two custom correlation rules (100010/100011, built during an earlier hands-on exercise) fire when the same source IP produces 3+ auth failures within a 2-minute window — a lower, more sensitive threshold than Wazuh's stock equivalents (8 in 120s). All figures below are pulled directly from the live OpenSearch indices (wazuh-alerts-*), not estimated.

READING THE ZERO
Zero correlated alerts isn't an absence of signal — it's perimeter controls (security groups, fail2ban) intercepting most attempts before they reach sshd's auth stage at all, and the nonexistent-user attempts that do land (10 over 30 days) haven't yet clustered tightly enough from a single source to cross the 3-in-120s threshold. That threshold is deliberately tuned to catch a genuine burst, not isolated scanning noise — this is the detector behaving correctly, not silently failing. (The real silent failure was somewhere else — see below.)
[ detection engineering: the real bug ]
FINDING
Rule 100011 — the correlation rule meant to catch repeated wrong-password attempts against a real username, arguably the more common brute-force pattern — has never actually been able to fire, since it was first written. Confirmed live during this report's own data-gathering, not assumed from the zero count above.
  1. ROOT CAUSE
    Rule 100011 depended on if_matched_sid=5716 — the stock rule for a generic "Failed" auth line. But Wazuh's engine always resolves an event to the deepest matching rule in the tree, and real "Failed password" log lines also match a more specific child rule, 5760. In practice, every genuine wrong-password attempt classified as 5760, never 5716 — so 100011's dependency was never satisfied.
  2. VERIFICATION, NOT ASSUMPTION
    Confirmed via wazuh-logtest against realistic log lines (not synthetic ones — OpenSSH's actual phrasing matters here): a plain "Failed password for <user>" line resolved to rule 5760, not 5716, exactly as suspected.
  3. FIX
    local_rules.xml · diff
      <rule id="100011" level="10" frequency="3" timeframe="120">
-   <if_matched_sid>5716</if_matched_sid>
+   <if_matched_sid>5760</if_matched_sid>
    <same_source_ip />
  4. RE-VERIFIED
    Fed 3 correlated failed-login events (same source, 20s apart) through wazuh-logtest against the patched ruleset — rule 100011 fired as designed: "Alert to be generated." Deployed via systemctl restart wazuh-manager, and backported into the lab's Ansible playbook (it had only ever been applied live over SSH — a rebuild would have silently lost it).

[ lessons learned ]
  • A rule that passes config syntax check and "looks" deployed isn't the same as a rule that's verified to actually fire — 100011 sat silently broken through the entire original exercise because nothing forced a real-log verification pass.
  • Wazuh's "deepest matching rule wins" behavior is a real gotcha when building correlation rules on top of stock sids — always trace which specific child rule a realistic log line actually resolves to, don't assume the parent sid you read about is the one that fires in practice.
  • Infra-as-code drift is invisible until you go looking: two custom detection rules existed only as live, unversioned changes on the box for over a day before this report's data-gathering surfaced and closed that gap.
