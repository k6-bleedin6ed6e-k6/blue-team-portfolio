[ phase 1 · triage ]
initial triage // 5 minutes
  • 01
    Verify alert in Splunk — confirm client_ip and domain volume 
    index=nextdns client_ip=<ALERT_IP>
| stats count by domain
| sort - count
| head 20
  • 02
    Check if IP is a known device (phone, laptop, tablet) against NextDNS client name registry
  • 03
    If known-good device → close as false positive and tune DET-001 threshold. Document the FP.
[ phase 2 · investigate ]
investigation // 15 minutes
  • 01
    Identify device by cross-referencing NextDNS client name with known inventory
  • 02
    Submit top domains to VirusTotal — look for known C2 infrastructure 
    index=nextdns client_ip=<ALERT_IP>
| stats count, avg(len(domain)) as avg_len by domain
| sort - avg_len
| head 10
  • 03
    Hunt for beaconing pattern — regular intervals indicate automated tooling 
    index=nextdns client_ip=<ALERT_IP>
| delta _time as gap
| stats avg(gap) by domain
| where avg(gap) > 300 AND avg(gap) < 3600
[ phase 3 · contain ]
containment // 5 minutes
  • 01
    Block domain in NextDNS denylist — immediate DNS-layer enforcement
  • 02
    Isolate device via router admin panel — remove network access pending investigation
  • 03
    Reimage device if compromise confirmed — do not attempt to clean; assume persistence
[ phase 4 · recover ]
recovery & documentation // 5 minutes
  • 01
    Document all IOCs — domain names, client IPs, timestamps, query volumes
  • 02
    Update DET-001 threshold based on investigation findings — reduce false positive rate
  • 03
    Add device to watchlist post-recovery — monitor for 48 hours after reintegration
